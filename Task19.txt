Лабораторная работа: Графовая база данных в MS SQL Server
Вариант 19: База знаний (KnowledgeBase) — Статьи, теги, категории, ссылки (связность контента)


Описание предметной области:

Необходимо создать графовую базу данных для хранения информации о структурированной базе знаний. База данных моделирует:

1. Статьи — информационные материалы базы знаний с уникальным идентификатором, заголовком, содержанием, датой создания, автором и количеством просмотров.
2. Теги — ключевые слова для маркировки статей с уникальным именем, описанием и счётчиком использования.
3. Категории — иерархические разделы для группировки статей с названием и ссылкой на родительскую категорию.
4. Ссылки между статьями — связи между различными статьями с указанием типа ссылки (internal, related, external), веса/важности связи (от 1 до 3) и даты создания ссылки.
5. Тегирование статей — присвоение тегов статьям с указанием даты присвоения тега и оценки релевантности (от 0 до 1).
6. Принадлежность к категориям — отнесение статьи к определённой категории с указанием признака основной категории (у статьи может быть только одна основная категория).

Задачи, решаемые с помощью графовой базы:

1. Найти все статьи, отмеченные определённым тегом (например, тегом "SQL") с сортировкой по популярности (количеству просмотров).
2. Найти статьи, принадлежащие указанной категории и её дочерним категориям (например, "Основы БД").
3. Построить граф ссылок между статьями с указанием типа ссылки и её веса.
4. Вычислить популярность каждого тега — количество статей, суммарное количество просмотров и среднее число просмотров на статью.
5. Найти статьи, на которые чаще всего ссылаются (топ-5 по входящим ссылкам).
6. Найти статьи с максимальным количеством исходящих ссылок (топ-5 по исходящим ссылкам).
7. Найти статьи, имеющие ссылки на статьи с указанным тегом (например, тегом "NoSQL").
8. Определить самую популярную статью в каждой категории (максимальное количество просмотров).
9. Найти связанные статьи через общие теги — определить пары статей, имеющих наибольшее количество общих тегов.
10. Построить данные для визуализации графа знаний — получить все связи между статьями с указанием веса для построения интерактивной схемы в Power BI.
